## Working Rules
1. Không code toàn bộ trong một lần.
2. Luôn chia nhỏ task.
3. Mỗi bước phải có:
   - Phân tích ngắn
   - Kế hoạch
   - Code
   - Kiểm tra
   - Sửa lỗi nếu có
4. Không hỏi lại nếu thông tin đã đủ.
5. Ưu tiên tạo website chạy được trước, tối ưu sau.
6. Không hard-code dữ liệu lớn trong component.
7. Toàn bộ nội dung chính đưa vào thư mục `config`.
8. Component phải nhỏ, rõ trách nhiệm.
9. Chạy build và sửa lỗi trước khi báo hoàn thành.
10. Sau mỗi task, báo lại file đã tạo/sửa.

## Code Standards
